What is Access Control?

Access Control is a security technique that regulates who can view or use resources in a computing environment or physical facility. In data centers, it's critical for protecting valuable equipment, data, and infrastructure.

Key Functions of Access Control:

  • Authentication: Verify the identity of individuals
  • Authorization: Determine access rights and privileges
  • Accountability: Track and log all access events
  • Administration: Manage users, credentials, and policies
  • Audit: Review access history and compliance

The Access Control Process: From credential presentation to access decision

Access Control Principles:

  • Least Privilege: Grant minimum necessary access
  • Separation of Duties: Distribute critical functions
  • Defense in Depth: Multiple security layers
  • Fail Secure: Default to locked state

Types of Credentials

Access control systems use various credential types, each with different security levels and applications.

1. Card-Based Credentials:

Proximity Cards (125 kHz)

  • Technology: Low-frequency RFID
  • Read Range: 2-8 inches typically
  • Security: Basic - no encryption
  • Cost: Low ($2-5 per card)
  • Application: Legacy systems, low-security areas

Smart Cards (13.56 MHz)

  • Technology: High-frequency RFID with chip
  • Read Range: 0-4 inches
  • Security: High - encryption and authentication
  • Cost: Medium ($5-20 per card)
  • Application: High-security areas, multi-application

2. Biometric Credentials:

Type Accuracy Speed User Acceptance Cost
Fingerprint High (FAR 0.001%) Fast (1-2 sec) Good Low-Medium
Facial Recognition Very High Very Fast (<1 sec) Excellent Medium-High
Iris Scan Highest (FAR 0.0001%) Fast (2-3 sec) Fair High
Hand Geometry Medium Fast (1-2 sec) Good Medium

3. Multi-Factor Authentication:

Combining Factors for Enhanced Security

  • Something You Have: Card, fob, mobile device
  • Something You Know: PIN, password
  • Something You Are: Biometric
  • Data Center Best Practice: Card + PIN or Card + Biometric
  • Critical Areas: Two or three factors required

Access Control System Components

1. Card Readers and Devices

Reader Types and Features:

  • Wall Mount Readers: Standard indoor/outdoor installation
  • Mullion Readers: Narrow door frame mounting
  • Desktop Readers: Enrollment and verification
  • Long-Range Readers: Vehicle and gate access
  • Mobile Readers: Smartphone-based credentials

2. Access Control Panels

  • Intelligent Controllers: Local decision making
  • Communication: TCP/IP, RS-485, Wiegand
  • Capacity: 2-64 doors per panel typically
  • Memory: Store credentials and transactions
  • Battery Backup: Maintain operation during power loss

3. Locking Hardware

Electric Lock Types:

  • Magnetic Locks (Maglocks): 600-1200 lbs holding force
  • Electric Strikes: Work with mechanical locksets
  • Electric Mortise Locks: High security, audit trail
  • Electric Panic Hardware: Emergency egress compliance
  • Electrified Cylindrical Locks: Retrofit applications

4. Request-to-Exit (REX) Devices

Device Type Technology Application Advantages
Motion Sensors PIR detection General exit Hands-free
Push Buttons Mechanical/Electronic Manual exit Deliberate action
Touch Bars Capacitive sensing High traffic ADA compliant
Emergency Release Break glass Emergency only Code required

5. Power Supplies

  • Centralized Power: Multiple locks from one supply
  • Distributed Power: Individual lock power supplies
  • Battery Backup: 4-24 hours typical
  • Fire Alarm Interface: Automatic unlock on fire
  • Monitoring: AC loss, battery status

System Integration

Access control systems must integrate with other security and building systems for comprehensive protection and operational efficiency.

1. Video Surveillance Integration

Key Integration Features:

  • Video Verification: Live view on access granted/denied
  • Transaction Search: Find video by access event
  • Visual Audit Trail: Paired video with access logs

3. Best Practices

‚úÖ Data Center Access Control Best Practices:

  • Implement anti-passback to prevent credential sharing
  • Use two-person rule for critical areas
  • Deploy mantrap/security vestibules at data halls

  • Implement time-based access restrictions
  • Regular access reviews and de-provisioning

4. Troubleshooting Common Issues

Common Problems and Solutions:

  • Card Read Failures: Check reader power, wiring, and card condition
  • Door Not Locking: Verify lock power, alignment, and REX sensors
  • Communication Loss: Check network connectivity and panel status
  • Database Errors: Verify server resources and connectivity
  • False Alarms: Adjust REX sensitivity and door contacts
